# Sun Life Financial Agent Scaffold

This repository now contains a small TypeScript agent core for a Sun Life insurance assistant focused on dental and orthodontic questions.

## What is included

- A typed `SunLifeFinancialAgent` service for answering user questions
- A pluggable `SunLifeRetriever` interface for RAG integrations
- Deterministic recommendation generation for local development
- An optional `LlmClient` interface if you want the final explanation generated by a model later
- A mock in-memory document retriever with sample Sun Life source records
- A formatted response shape that matches the booth UX specification

## Project structure

- `finance/agent.ts`: main orchestration logic
- `finance/types.ts`: shared types for request, retrieval, and response models
- `finance/retriever.ts`: in-memory retriever implementation
- `finance/mockData.ts`: sample Sun Life source documents
- `finance/prompt.ts`: system prompt and response formatter
- `finance/examples/demo.ts`: simple example entry point

## Quick start

1. Install dependencies:

```bash
npm install
```

2. Run type-check:

```bash
npm run check
```

3. Run the demo:

```bash
npm run demo
```

## Integration notes

For production use, replace `InMemorySunLifeRetriever` with a retriever backed by:

- a vector database of Sun Life PDFs and webpages
- a CMS or document ingestion pipeline
- a search service restricted to Sun Life sources

You can also attach an `llm` implementation to the agent if you want the natural-language explanation to be generated dynamically after retrieval. The recommendation must still stay grounded in the retrieved Sun Life documents.

## Important limitation

The sample source records in `finance/mockData.ts` are placeholders for scaffolding and integration only. Before shipping, replace them with real Sun Life documentation metadata and excerpts gathered through your RAG pipeline.
